# 小说 Lint 规则目录

> **唯一数据源**：所有规则 ID、阈值、严重级别以本文件为准。禁止在其他文件维护同一条规则的不同版本。
>
> **配合使用**：本规则目录是 novel-writing 技能的一部分，依赖其目录结构（`novel-memory/`）和模板定义。

---

## 执行模式

| 模式 | 触发场景 | 执行规则 | 输出 |
|------|---------|---------|------|
| paragraph | 快速审核单段 | `PR-*` | 该段 error/warn/info + 修复建议 |
| chapter | 定稿前完整审核 | `CH-*` + `PR-*` 抽样 | PASS / PASS_WITH_WARNINGS / BLOCKED |
| repo | 整卷/全书巡检 | 全部规则按卷批量 | 按规则 ID 聚合的统计报表 |

### 门禁策略

- `error > 0` → **BLOCKED**（不可宣称通过）
- `error = 0 且 warn > 0` → **PASS_WITH_WARNINGS**
- `error = 0 且 warn = 0` → **PASS**

### AI 味检测（PR-AI）

> 规则来源：`references/去AI味指南.md`（删除清单 + 三条原则 + 风险模式速查表）。
> **核心理念**：修复建议不给具体替换方案（会变成新模板），只要求"删除"或"自行决定改法且不重复"。

| ID | Severity | 检查项 | 失败条件 | 修复建议 |
|----|----------|--------|----------|----------|
| PR-AI-001 | warn | 高频废词重复 | 同一废词在**同一章**出现 ≥2 次 | 删除多余实例，不指定替换方案 |
| PR-AI-002 | warn | 句式起始重复 | 连续 3 句同主语/同句式开头 | 自行变换，不重复上次的改法 |
| PR-AI-003 | warn | 抽象词堆叠 | 抽象情绪词连续堆叠 | 删除 |
| PR-AI-004 | warn | 概述句代替展示 | 出现「他感到…」「她意识到…」等 tell-not-show 句式 | 删除 |
| PR-AI-005 | warn | 伪深刻总结句 | 章末/段末出现「这一刻，他终于明白了…」类句式 | 删除 |
| PR-AI-006 | warn | 量词模式重复 | 同一「一+量词」模式在**同一章**出现 ≥2 次 | 删除多余实例 |
| PR-AI-007 | warn | 程式化情绪标签 | 出现「满脸不可置信」「眼中闪过一丝X」等标签 | 删除 |
| PR-AI-008 | warn | 模糊概述+比喻组合 | 模糊概述句紧跟装饰性比喻 | 两句都删 |
| PR-AI-009 | warn | 能力旁白解说 | 旁白直接定义角色能力机制 | 删除 |
| PR-AI-010 | warn | 章尾三件套 | 章末出现「独处反思→总结收获→内心立志」模式 | 删末尾三段，用动作收尾 |
| PR-AI-011 | warn | 指令播报链过长 | 连续 ≥3 段「汇报→下令→确认」 | 合并为间接叙述，保留关键对话 |
| PR-AI-012 | warn | 术语密度过高 | 300 字窗口同域术语 ≥8 次且缺 POV 观察 | 削减术语，自行决定改法 |
| PR-AI-013 | warn | 解释句连发 | 200 字内 ≥3 句「这意味着/这说明/如果…」 | 删解释句 |
| PR-AI-014 | warn | 结尾收束过满 | 章末连续 2 段收口，无未解细节 | 删末段总结 |
| PR-AI-015 | warn | 感官堆叠公式 | 同章 ≥2 次「X味混着Y味+身体反应」句式 | 删除重复实例 |
| PR-AI-016 | warn | 体感微反应均一化 | 情绪编码全靠同一套体感（手指收紧/指节泛白/呼吸变化），同章同一体感 ≥2 次 | 删除重复，自行换不同路径 |
| PR-AI-017 | warn | 数字密度过高 | 500 字内精确数字（含百分比） >3 个 | 保留关键数字，其余模糊化 |
| PR-AI-018 | warn | 术语即时投喂 | 同场景"术语+即时解释句" ≥2 组 | 删解释句，让读者推断 |
| PR-AI-019 | warn | 句号密度过高 | 连续 ≥4 个短句（≤8 字）用句号断开 | 用逗号合并部分短句，但不要整段串成逗号长链；目标是长短交替 |
| PR-AI-020 | warn | 排比结构 | 三连排比（「X不对，Y不对，Z也不对」）| 打散结构 |
| PR-AI-021 | warn | 氛围式短句收尾 | 章/场景末尾以两个极短氛围句收束（「风吹过。他没动。」）| 换成具体动作收尾 |
| PR-AI-022 | warn | 渐进式环境描写 | 「先是X…再往前Y…然后Z…」式递进描写环境变化 | 打散递进结构，中间插入对话/动作/闲笔 |

## 3) 严重级别与升级准则

- **error**：确定性、可证明、低争议，适合门禁阻断。
- **warn**：启发式、存在少量误报，不阻断。
- **info**：质量建议，仅用于提示。

### 升级流程

1. 新规则先上线 `warn`。
2. 连续两轮人工复核误报率 < 5% 后，再考虑升为 `error`。
3. 升级时必须记录变更原因与样本依据。

## 5) 与工作流的集成点

| 工作流阶段 | 执行的 lint 规则 | 执行者 |
|-----------|-----------------|--------|
| 审核阶段 文笔审稿人 | `PR-*` 全部（含 `PR-CH-*`、`PR-DLG-RATIO-*`） | 文笔审稿子代理 |
| 录入阶段 合规审查 | `CH-PATH-*` + `CH-TPL-*` + `CH-FM-*` + `CH-SYNC-*` + `CH-CHECK-*` | 合规审查子代理 |
| 卷审阶段 | repo 模式全部规则 | 卷审子代理 |
