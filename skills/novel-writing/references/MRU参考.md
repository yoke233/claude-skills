# MRU 参考（动机-反应单元）

来源：Dwight Swain《Techniques of the Selling Writer》

## 核心概念

MRU（Motivation-Reaction Unit）是小说写作的最小叙事单元。每一个「事件」都由一个**动机**（外部刺激）和一个**反应**（角色回应）组成。

---

## 结构

```
动机（Motivation）= 角色之外发生的事
        ↓
反应（Reaction）= 角色的回应，严格按以下顺序：
    1. 感受（本能/情感反应）
    2. 动作（身体反应）
    3. 对话（理性反应/言语）
```

---

## 反应顺序铁律

**感受 → 动作 → 对话**，不可打乱。

**为什么？** 因为人的真实反应就是这个顺序：先感受到冲击，身体本能反应，然后才来得及说话。

### 正确示例

> 一把寒光闪过他面前（**动机**）。他心头一跳（**感受**），猛地侧身后退（**动作**）。「谁！」他厉声喝道（**对话**）。

### 错误示例

> 一把寒光闪过他面前。「谁！」他大喊（❌ 对话先于感受和动作）。他心里有点害怕，赶紧退后一步。

---

## 动机的写法

### 原则

- 动机必须是角色能感知到的**外部事件**
- 用客观描述，不带角色的主观判断
- 动机要足够具体，让读者能「看到」

### 正确

> 大门轰然炸开，碎木四溅。

### 错误

> 门突然被一股可怕的力量摧毁了。（「可怕的」是角色的主观判断，应该放在反应部分）

---

## 反应的写法

### 感受（第一层）

本能、瞬间的情绪反应，通常只需一两句。

```
心头一紧 / 一股寒意 / 血液涌上头顶 / 胃里翻涌
```

**注意**：感受不需要用大段心理描写。越短促越真实。

### 动作（第二层）

身体的反应——战斗、逃跑、僵住。

```
猛地转身 / 握紧剑柄 / 下意识后退 / 挡在同伴前面
```

### 对话（第三层）

经过短暂思考后的言语反应。

```
质问 / 求助 / 威胁 / 安慰 / 解释
```

---

## 省略的艺术

不是每个 MRU 都需要完整三层。在快节奏场景中可以省略：

| 节奏 | 省略方式 |
|------|----------|
| 极快（战斗） | 只写动作，省略感受和对话 |
| 快速 | 写感受+动作，省略对话 |
| 正常 | 完整三层 |
| 慢速 | 三层都展开，感受部分详写 |

---

## 常见错误

| 错误 | 纠正 |
|------|------|
| 对话先于感受 | 人不可能被吓到后先开口再害怕 |
| 动机中夹杂主观判断 | 动机写客观事实，判断放到反应中 |
| 反应没有动机 | 角色突然做出反应但读者不知道为什么 |
| 动机太模糊 | 「感觉到危险」不如「脖颈汗毛竖起」 |
| 所有反应都是完整三层 | 快节奏时适当省略 |

---

## 练习方法

写完每个场景后，检查：

1. 每个反应之前是否有明确的动机？
2. 反应的顺序是否正确（感受→动作→对话）？
3. 快节奏场景的 MRU 是否足够精简？
4. 慢节奏场景的 MRU 是否足够展开？

## 展示代替叙述

### 概述句代替场景展示
- **错误**：「他感到一阵强烈的愤怒。」
- **正确**：「他的指节捏得发白，牙关磨出了声响。」

### 旁白式情感解读
- **错误**：「她的眼神中流露出一丝不易察觉的忧伤。」
- **改写策略**：让另一个角色注意到这个细节，或通过行为暗示。

### 能力/设定旁白解说
- AI 喜欢在角色展现能力后，用旁白直接解释机制：「这就是他的武意。受伤越重，战力越强。」
- 读者已经在战斗中**看到**了效果，不需要旁白再**说**一遍
- **规则**：能力机制只能通过角色体验、对话、或敌人的惊讶反应来间接传达，禁止旁白直接定义

## 对话技巧

### 对话标签

#### 禁止模式
- 频繁使用"XX说道""XX回答道""XX解释道"——动词无差别
- 每句对话都带标签

#### 正确模式
- 标签动词要有差异：低声说/吼道/嗤了一声/没接话
- 2-3 句对话后才加一次标签，中间用动作拍（action beat）区分说话人
- 动作拍示例：「沈渊把刀插回鞘里。"走吧。"」——不需要"沈渊说"

### 角色对话辨识度
- AI 生成的角色缺乏辨识度：所有人说话节奏一样，情绪反应一样，内心独白用同一种句式
- **检测方法**：遮住角色名，读一段对话，能否分辨是谁在说话？不能 → 需要重写
- **纠正**：给每个角色分配独有的语言习惯（口头禅、句长倾向、用词层次），在角色模板「说话风格」字段中明确标注

## 感官描写

AI 写作严重偏向视觉描写，常见缺失：

| 感官 | 检查点 |
|------|--------|
| 嗅觉 | 战斗场景有血腥味吗？荒原有灰土气味吗？ |
| 触觉 | 握武器的手感？温度变化？疼痛具体位置？ |
| 听觉 | 不要只写"一声巨响"，写具体声响（骨头碎裂/布料撕裂/风过刀刃） |
| 味觉 | 嘴里的铁锈味、沙子味、干渴感 |
| 本体感觉 | 肌肉酸痛、关节发紧、重心偏移 |

**规则**：每个场景至少覆盖 3 种感官，视觉之外至少 2 种。

## 附录：写作修正技法

### 三层叠加法
用**场景层 + 情绪层 + 细节锚点**替代扁平描述：
- **扁平写法**：「雨夜，他感到很孤独。」
- **三层叠加**：「出租屋的窗帘没拉（场景），手机屏幕亮了一下又暗了，没人发消息（情绪暗示），雨水顺着晾衣绳滴在空啤酒罐上，叮、叮（细节锚点）」

### 做减法原则
- 好作者敢于省略。删掉读者能自己推断的内容
- **冗余写法**：「他听到这个消息后，感到非常震惊和愤怒，双手不自觉地握紧了拳头，指节发出咯咯的声响。」
- **减法版**：「他的拳头攥紧了。」——震惊和愤怒让读者自己感受

### 留白与不完美
- 真实写作有粗糙感：句子可以不完整，对话可以被打断，描写可以只给局部
- 每段"完美收束"反而不自然
- **纠正**：允许段落断在动作中间；对话可以"……"结尾不给答案；场景可以只描写一个角落而非全景
