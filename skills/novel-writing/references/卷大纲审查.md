# 卷大纲审查（检查点 B）

> **触发时机**：阶段〇·五 Step 7（卷大纲）完成后，进入章节写作之前。
>
> **目的**：卷大纲是 30 章的执行蓝图。写作一旦开始，修改大纲的代价极高。此处是最后的把关机会。

---

## 执行方式

使用 Agent 工具**并行启动 3 个审查子代理**。每个审查员只有 Read/Grep/Glob 权限，不修改任何文件。

---

## 审查员 1：技法审查

**需读取的文件**：
- 当前卷大纲
- `references/冲突类型手册.md`
- `references/伏笔类型与手法.md`
- `references/悬念技法手册.md`
- `references/节奏控制手册.md`
- `references/Scene-Sequel模型.md`
- `references/爽点虐点设计手册.md`

**检查清单**：

### T1：冲突设计
- [ ] 主线冲突有 ≥2 层级（表层+深层）？
- [ ] 使用了 ≥2 种冲突类型？
- [ ] 冲突有升级阶梯（不是一个强度打到底）？
- [ ] 高潮处冲突有代价？

### T2：伏笔规划
- [ ] 每条伏笔埋设时有当前场景功能？（不是硬塞）
- [ ] 同时活跃伏笔 ≤5 条？
- [ ] 所有本卷埋设的伏笔都有回收计划（可跨卷）？
- [ ] 前卷遗留的活跃伏笔有强化安排？

### T3：钩子分布
- [ ] 无连续 3 章以上使用同一类型钩子？
- [ ] 强钩子和缓钩子有交替？（大约 1 强 + 2 缓的节奏）
- [ ] 每一章都有钩子安排？（没有遗漏章）

### T4：节奏曲线
- [ ] 无连续 3+ 章高张力？（读者疲劳）
- [ ] 无连续 3+ 章低张力？（读者无聊）
- [ ] 高潮在本卷约 3/4 处？
- [ ] Scene 和 Sequel 有交替？（不是全 Scene 或全 Sequel）

### T5：情绪事件
- [ ] 爽点 3-5 个/卷？
- [ ] 虐点 1-2 个/卷？
- [ ] 虐点后 3 章内有希望？
- [ ] 每个爽点前有 ≥3 章铺垫？
- [ ] 爽点没有连续出现？

### T6：叙事装置
- [ ] 至少使用了 1 种叙事装置？
- [ ] 装置使用不超过 3 种？（过多则花哨）

**输出格式**：
```markdown
## 技法审查

| 检查项 | 结果 | 说明 |
|--------|------|------|
| T1 冲突 | PASS/FAIL | |
| T2 伏笔 | PASS/FAIL | |
| T3 钩子 | PASS/FAIL | |
| T4 节奏 | PASS/FAIL | |
| T5 情绪 | PASS/FAIL | |
| T6 装置 | PASS/FAIL | |

### 必须修复
- ...

### 建议调整
- ...
```

## 审查员 2：设定一致性审查

**需读取的文件**：
- 当前卷大纲
- `10-世界观/概述.md` + 相关子文件
- `20-角色/` 下本卷出场角色文件
- `30-情节/主线.md`

**检查清单**：

### S1：章节事件 vs 世界规则
- [ ] 逐章检查：每章的关键事件是否符合已建立的世界规则？
- [ ] 角色在大纲中展示的能力是否在其当前修为范围内？
- [ ] 地点描述与世界观地理设定是否一致？

### S2：角色行为 vs 人设
- [ ] 大纲中每个角色的行为决策是否符合其性格和动机？
- [ ] 角色之间的互动是否符合已建立的关系？
- [ ] 有没有角色做出"剧情需要但不符合人设"的行为？

### S3：时间线可行性
- [ ] 大纲中各事件的时间间隔是否合理？
- [ ] 角色在场景之间的移动时间是否合理？（不能"瞬移"）
- [ ] 修炼/恢复/旅行等需要时间的事件，时间分配是否充足？

### S4：前卷衔接（非第一卷时检查）
- [ ] 前卷结尾的状态与本卷开头是否衔接？
- [ ] 前卷遗留的悬念/伏笔在本卷是否有处理计划？
- [ ] 角色的修为/状态是否从前卷正确继承？

**输出格式**：
```markdown
## 设定一致性审查

| 检查项 | 结果 | 说明 |
|--------|------|------|
| S1 事件vs规则 | PASS/FAIL | 第X章有矛盾 |
| S2 行为vs人设 | PASS/FAIL | |
| S3 时间线 | PASS/FAIL | |
| S4 前卷衔接 | PASS/SKIP | |

### 必须修复
- ...
```

## 审查员 3：读者体验审查

**需读取的文件**：
- 当前卷大纲（重点：章节概要表、张力曲线、情绪事件）
- `00-项目/项目总览.md`（了解目标读者）

**检查清单**：

### R1：开篇吸引力（前 3 章）
- [ ] 第 1 章是否在 500 字内建立核心悬念或展示最独特的设定？
- [ ] 前 3 章是否完成"黄金三章"（角色认同 + 核心冲突 + 阅读承诺）？
- [ ] 前 3 章有没有过多设定说明？（读者还没 care 的时候不要灌设定）

### R2：粘性检查
- [ ] 有没有连续 5+ 章没有任何爽点/燃点/悬念推进？
- [ ] 每 10 章左右是否有一个明确的"阶段性满足"？
- [ ] 信息揭示是否有节奏？（不能一次全说完，也不能吊太久）

### R3：高潮设计
- [ ] 本卷高潮是否是全卷张力最高点？
- [ ] 高潮前的铺垫是否充分？（有没有"突然高潮"的感觉）
- [ ] 高潮后是否有收束段？（不能高潮完立刻结束）

### R4：卷尾留存
- [ ] 本卷结尾是否既有满足感又留有悬念？
- [ ] 下一卷的吸引力钩子是否已埋下？
- [ ] 读者读完本卷后是否有明确理由继续读下一卷？

**输出格式**：
```markdown
## 读者体验审查

| 检查项 | 结果 | 说明 |
|--------|------|------|
| R1 开篇 | PASS/WARN/FAIL | |
| R2 粘性 | PASS/WARN/FAIL | |
| R3 高潮 | PASS/WARN/FAIL | |
| R4 留存 | PASS/WARN/FAIL | |

### 必须修复（FAIL）
- ...

### 建议改善（WARN）
- ...
```

---

## 综合评判与处理

汇总 3 个审查员的结果：

- **有 FAIL** → 修改卷大纲后重新审查（只重审修改涉及的审查员）
- **只有 WARN** → 由用户决定是否修复，记录到 `00-项目/待解决.md`
- **全 PASS** → 进入阶段一（章节写作）
