# 去 AI 味指南

AI 文本可检测，根因只有一个：**均匀和可预测**。不是因为用了某个词或某种技法，而是因为任何技法都被 AI 以相同方式、相同密度、相同节奏施用于全文。规则越多、纠正方案越具体，AI 执行得越整齐，检测器越好抓。因此本指南只做两件事：列出必删的垃圾，然后给三条不可能被"均匀执行"的原则。

---

## 一、删除清单（出现即删，无需替换）

以下内容出现就删掉。不提供替换方案——替换方案会变成新模板。删掉之后那个位置留空白、换成对话、写动作、或者什么都不写，随意。

### 废词
- 微微、缓缓、淡淡、不禁、忍不住、情不自禁
- 心中暗想、心中一动、内心深处、一股莫名的感觉涌上心头
- 与此同时、就在这时、不由自主地、下意识地
- 深邃的眼眸、锐利的目光、清冷的气质

### 废句式
- **伪深刻总结**：「这一刻，他终于明白了什么是真正的力量。」——删。
- **模糊概述 + 装饰比喻**：「那是一种说不出来的感觉。像是一把钝了的刀，突然开了刃。」——两句都删。
- **程式化情绪标签**：满脸不可置信、眼中闪过一丝惊讶、嘴角微微上扬——删。
- **章尾三件套**：①独处反思 → ②旁白总结收获 → ③内心立志/感慨——把最后三段删掉，章节仍完整则说明这三段是惯性收尾。

### 量化上限
- 同一个废词/废句式：同章 ≤ 1 次
- 同一「一+量词」模式（一股/一阵/一道/一抹）：同章 ≤ 1 次
- 破折号：每章 ≤ 5 个
- 过渡词（然而/不过/随即/紧接着）：每段 ≤ 1 个
- 连续同主语/同句式开头：≤ 2 句

## 二、三条原则

不给具体做法。具体做法会变成模板。只给方向，每次怎么做必须不一样。

### 原则一：不均匀

**AI 的本质缺陷是均匀。** 句长均匀、情绪编码均匀、描写密度均匀、信息投喂节奏均匀。人类写作天然不均匀——会突然写一段很长的话，也会连着三个字就换行；会连续对话不插叙述，也会连续三段描写没有一句对话。

**执行标准**：任何可识别的"手法"（体感微反应、感官描写、短句、数字、术语解释……），连续用了两次，第三次必须换完全不同的方式。不限定换成什么——只要和前两次不一样。

**自检**：把连续 5 段的首句类型标出来（动作/描写/对话/心理/闲笔），如果呈现规律交替，就是机械循环，必须打乱。

### 原则二：留缺口

**AI 的本质冲动是解释一切。** 每个设定即时交代，每个情绪即时编码，每个数字精确给出，每个因果链闭合。人类小说大量信息是读者自己补的——不解释的地方比解释的地方多。

**执行标准**：
- 每 500 字至少有 1 处读者需要自行推断的因果关系
- 术语直接用，不即时解释；读者看不懂就看不懂，后面自然会懂（或永远不懂）
- 精确数字是稀缺资源，只在关键转折点给；其余用模糊表达
- 章末必须有未解的信息缺口，禁止收束干净

### 原则三：加噪声

**AI 的本质倾向是每个细节都服务于情节/主题/氛围。** 人类写作有大量"无意义"的细节——鞋带松了、墙皮在掉、杯子上有个缺口——这些东西不象征任何事、不推进任何情节、不暗示任何伏笔，纯粹是人类视角的噪声。AI 不会主动写这些，因为它觉得"没用"。但正是这些噪声让文本像人写的。

**执行标准**：每 500 字至少插入 1 个与情节、主题、象征完全无关的具体细节。这个细节必须是该 POV 角色在当前位置会随机注意到的东西，但对故事毫无意义。

## 三、风险模式速查（只诊断，不开药方）

以下是检测器敏感但不属于"废词废句"层面的结构性问题。识别到就改，但怎么改自己决定——不要每次用同一种改法。

| 症状 | 识别方法 | 为什么会被检测 |
|------|---------|--------------|
| **播报链** | 连续 ≥ 3 段「汇报→下令→确认」| 流程在重复，人物关系没推进 |
| **感官堆叠公式** | 同章 ≥ 2 次「X味混着Y味+身体反应」| 结构模板重复 |
| **体感微反应均一化** | 情绪全靠手指收紧/指节泛白/呼吸变化 | 纠正方案本身变成了新模板 |
| **数字密度过高** | 500 字内精确数字（含百分比） > 3 个 | 像系统日志不像叙事 |
| **术语即时投喂** | 术语出现后紧跟解释句 | 零认知摩擦 = AI 特征 |
| **解释句连发** | 200 字内 ≥ 2 句「这意味着/这说明/如果…」| 作者替读者推理 |
| **单句修饰词过载** | 单句形容词+副词 ≥ 3 个 | 华美而空洞 |
| **段段起承转合** | 每段都是"引入→展开→收束"完整小作文 | 太整齐 |
| **句号密度过高** | 连续多个短句用句号断开（「X了。Y了。Z了。」） | 机械断句，节拍器节奏；适当用逗号合并，但不要矫枉过正串成逗号长链 |
| **渐进式环境描写** | 「先是X…再往前Y…然后Z…」式递进结构 | 环境变化太有序，像导游解说词 |
| **段落密度过高（单句独段）** | 连续 ≥5 个段落都是 1-2 句话，每段空行分隔 | 匀速短段锤击；同一时刻的动作/观察/对话应合进同一段 |
| **纯描写段过长** | 连续 ≥150 字的叙述/描写/动作段中对话 ≤1 句 | 缺人声 = AI 特征；优先砍短描写段，或插入非功能性闲话（禁止问→答/命令→确认/术语→解释） |

## 四、通过检测的文本长什么样

> 从朱雀检测低分样本（AIGC < 0.2）和经验证的改写样本提取的共性特征。不是规则，是参照物。

- **对话占比高且节奏不规则**：有人说半句话、有人不回答、对话之间不一定插叙述；用日常闲话当噪声（"还有炭笔吗"）
- **间接叙述压缩播报**：把多轮"汇报→下令→确认"合并成一段间接叙述（"呼吸塔的人说……工坊那边说……"），只保留关键对话
- **信息留白**：不解释"为什么"，只呈现"发生了什么"；术语直接用不解释（"报了个偏角"）
- **非功能性细节**：与情节无关的具体观察——断腿椅子、墙角水渍、手套蹭完裤腿更黏了
- **逗号合并但不过度**：避免"X了。Y了。Z了。"的句号密度，适当用逗号合并，但不要把整段串成一条逗号长链（读不下去且形成新的均匀模式）；关键是长短句交替——短句喘气、长句推进，节奏不均匀
- **动作收尾代替氛围收尾**：不以"风吹过。他没动。"式短句留白结尾，而是用一个具体动作收（"她站起来拍了拍膝盖上的碎石渣子"）
- **段落合并同一时刻**：同时发生的动作/观察/反应合进同一段，不要每句话独占一段；段落长短交替形成不均匀节奏（一个大段 ↔ 一个短段），而非匀速单句独段
- **描写段必须有人声**：连续描写/动作超过 150 字就必须插入对话，但对话必须是非功能性的（闲话、嘟囔、半句话），不能是问→答、命令→确认、术语→解释——这些功能性对话会触发术语投喂和播报链模式反而加分；间接叙述也算人声；最有效的方法往往是直接砍短描写段

## 五、自查清单

### 删除层
- [ ] 废词/废句式已清除，无遗漏
- [ ] 量化上限未超标

### 不均匀
- [ ] 连续 5 段首句类型不呈规律交替
- [ ] 同一手法未连续使用 ≥ 3 次
- [ ] 存在长短句明显交错，不是匀速节奏

### 留缺口
- [ ] 每 500 字 ≥ 1 处信息留白
- [ ] 无"术语+即时解释"连发（同场景 ≤ 1 次）
- [ ] 精确数字稀疏（500 字内 ≤ 3 个）
- [ ] 章末有未解信息缺口

### 加噪声
- [ ] 每 500 字 ≥ 1 个非功能性细节
- [ ] 这些细节不服务于情节/主题/象征

### 段落与描写
- [ ] 无连续 ≥5 个单句独段
- [ ] 无 ≥150 字纯描写段（无对话）
- [ ] 插入的对话非功能性（非问答/非命令确认/非术语解释）
