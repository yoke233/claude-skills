# 设定整合审查（检查点 A）

> **触发时机**：阶段〇·五 Step 1-6 完成后（项目定义、世界观、角色、情节、全书大纲、风格指南全部就位），进入卷大纲之前。
>
> **目的**：验证各设定零件之间是否咬合。单个零件可能看起来没问题，但组合后可能出现矛盾、冗余或缺失。

---

## 执行方式

使用 Agent 工具**并行启动 3 个审查子代理**。每个审查员只有 Read/Grep/Glob 权限，不修改任何文件。

---

## 审查员 1：逻辑一致性审查

**需读取的文件**：
- `10-世界观/概述.md` + 所有子文件（力量体系、地理、历史、社会、规则）
- `35-时间线/世界历史线.md`

**检查清单**：

### L1：力量体系自洽
- [ ] 各等级之间的差距是否明确定义？（不能出现"看情况"的模糊地带）
- [ ] 突破条件是否每个等级都有？
- [ ] 有没有某个能力在设定中被描述为"罕见"，但多个角色都拥有？
- [ ] 力量上限是否明确？（防止后期战力崩坏）

### L2：地理与历史一致
- [ ] 地图上的距离与历史事件中的行军/旅行时间是否匹配？
- [ ] 各势力的地盘与其实力是否匹配？
- [ ] 资源分布与经济/冲突逻辑是否自洽？

### L3：规则无矛盾
- [ ] 世界规则之间有没有互相打架的？（例：设定说"灵气稀薄万物凋零"，但某地却有繁茂森林）
- [ ] 社会规则与力量体系是否兼容？（例：设定说"强者为尊"，但社会结构是官僚体制）

### L4：时间线可行
- [ ] 世界历史线的事件先后顺序是否合理？
- [ ] 主角年龄与经历的时间跨度是否匹配？

**输出格式**：
```markdown
## 逻辑一致性审查

| 检查项 | 结果 | 说明 |
|--------|------|------|
| L1 力量体系 | PASS/WARN/FAIL | |
| L2 地理历史 | PASS/WARN/FAIL | |
| L3 规则矛盾 | PASS/WARN/FAIL | |
| L4 时间线 | PASS/WARN/FAIL | |

### 必须修复（FAIL）
- ...

### 建议修复（WARN）
- ...
```

## 审查员 2：角色-剧情匹配审查

**需读取的文件**：
- `20-角色/` 下所有角色文件
- `20-角色/_角色索引.md`
- `30-情节/主线.md`
- `40-大纲/全书大纲.md`

**检查清单**：

### CP1：角色必要性
- [ ] 每个已创建的角色在主线或副线中是否都有明确作用？
- [ ] 有没有功能重叠的角色？（两个角色做同一件事）
- [ ] 有没有情节需要某种角色但还没创建的？

### CP2：动机-情节驱动
- [ ] 主角的核心动机是否直接驱动主线冲突？
- [ ] 反派/对手的动机是否与主角形成正面碰撞？（不能是平行线）
- [ ] 每条活跃情节线是否都有对应的角色在推动？

### CP3：角色弧线可行性
- [ ] 主角的起始状态和全书大纲规划的结束状态之间，变化幅度是否合理？
- [ ] 角色弧线是否有足够的事件来支撑？（不能"突然顿悟"）
- [ ] 核心角色之间的关系变化是否有计划？

### CP4：角色能力与情节匹配
- [ ] 情节中需要角色做的事，角色的能力是否支持？
- [ ] 有没有角色拥有某种能力但全书从不使用？（契诃夫之枪反面）
- [ ] 主角的初始能力是否足以启动第一卷的冲突？

**输出格式**：
```markdown
## 角色-剧情匹配审查

| 检查项 | 结果 | 说明 |
|--------|------|------|
| CP1 角色必要性 | PASS/WARN/FAIL | |
| CP2 动机驱动 | PASS/WARN/FAIL | |
| CP3 弧线可行性 | PASS/WARN/FAIL | |
| CP4 能力匹配 | PASS/WARN/FAIL | |

### 必须修复（FAIL）
- ...

### 建议修复（WARN）
- ...
```

## 审查员 3：可写性审查

**需读取的文件**：
- `00-项目/项目总览.md`
- `40-大纲/全书大纲.md`
- `10-世界观/概述.md`
- `10-世界观/力量体系/`（如存在）
- `30-情节/主线.md`

**检查清单**：

### W1：体量匹配
- [ ] 目标字数与分卷计划是否匹配？（每卷 20-30 章 × 3000-5000 字/章）
- [ ] 情节密度能撑到目标字数吗？（有没有"到第 50 章就没东西写了"的风险）
- [ ] 升级空间是否均匀分布在各卷？（不能前 2 卷升完，后面无事可做）

### W2：冲突可持续性
- [ ] 核心冲突能否持续到全书结束？（还是在某一卷就会被解决）
- [ ] 有没有备用冲突源？（主线解决后还有什么能撑住故事）
- [ ] 反派/对手的威胁级别能否随主角成长而升级？

### W3：新鲜感储备
- [ ] 世界观是否留有足够的未探索区域？（新地图、新势力、新规则的空间）
- [ ] 力量体系的上层是否有足够的设计空间？
- [ ] 情节线是否有分支可能？（不是一条直线走到底）

### W4：卡点预警
- [ ] 有没有可预见的"写死"风险？（设定过于封闭、主角太强太早、缺少有效对手）
- [ ] 全书大纲中有没有"待定"或"再说"的模糊区？超过 2 处则标记风险

**输出格式**：
```markdown
## 可写性审查

| 检查项 | 结果 | 说明 |
|--------|------|------|
| W1 体量匹配 | PASS/WARN/FAIL | |
| W2 冲突持续 | PASS/WARN/FAIL | |
| W3 新鲜感 | PASS/WARN/FAIL | |
| W4 卡点预警 | PASS/WARN/FAIL | |

### 必须修复（FAIL）
- ...

### 建议修复（WARN）
- ...
```

---

## 综合评判与处理

汇总 3 个审查员的结果：

- **有 FAIL** → 必须修复后重新审查（只重审修改涉及的审查员）
- **只有 WARN** → 由用户决定是否修复，记录到 `00-项目/待解决.md`
- **全 PASS** → 进入 Step 7（卷大纲）
