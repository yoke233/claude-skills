# 写作阶段

**触发**：用户说「写第X章」「继续写」或类似意图。

**架构**：主 agent 只做调度，不读重内容。搜集员、写手、审稿人通过磁盘文件交接。

## 章节写作的"双通道"原则（更自然，但不放水）

本技能把"写得顺"和"写得对"拆开处理：

- **快写通道（写手）**：优先叙事流、情绪推进、钩子；只守住"防翻车指令"等硬约束，避免写作时被规则感拖慢。
- **审核通道（审稿 + 合规）**：用多人审核（剧情/技法/角色/文笔）+ Lint + 写后即录合规审查把跑偏拉回去。

**硬承诺**：任何导出到 `manuscript/` 的文本，都必须经过审核阶段（多人审核）+ 录入阶段（写后即录）+ ★ 合规审查。写作可以快，但交付必须稳。

## 黄金三章：前三章是生死线

> **核心认知**：网文/连载的读者流失集中在前三章。70% 以上弃书发生在第一章前 1000 字内。第一章不是"铺垫章"，是"生存章"。

**全书第 1 章与新卷开篇的张力下限**：

| 章节位置 | 张力下限 | 说明 |
|----------|----------|------|
| 全书第 1 章 | ★★★★☆ | 不是"中等开场"，必须一开场就制造紧迫感 |
| 全书第 2-3 章 | ★★★☆☆ | 可以略降，但必须在第 3 章结束前完成"黄金承诺" |
| 非首卷的新卷开篇 | ★★★☆☆ | 允许中等张力，因读者已有沉没成本 |

> 如果卷大纲中第 1 章张力低于 ★★★★☆，搜集员在组装上下文包时**必须标注覆写**，将张力提升至下限。

**前 300-500 字的四项必答（所有章节通用）**：
1. **目标**：主角此刻要什么？
2. **阻碍**：现在被什么挡住？
3. **代价/时钟**：不解决会怎样？有什么明确后果或死线？
4. **读者问题**：读者此刻最想继续追问的一件事是什么？

**黄金三章附加要求（仅全书前 3 章 / 新卷前 2 章）**：

5. **即时共情**：前 500 字内读者必须对主角产生情感反应（同情/敬佩/好奇），不能只是"看一个陌生人执行任务"。手段包括：展示不公待遇、暴露脆弱面、呈现高压下的克制、让读者看到角色"有可能失去的东西"。
6. **世界观减负**：前 3 章每个场景最多 1 条设定信息（已有规则），但第 1 章**前 500 字内禁止引入超过 2 个新术语**。让读者先在乎人，再理解系统。
7. **钩子前置**：全书第 1 章的核心悬念/威胁**必须在前 1/3 篇幅内至少暗示一次**，不能全部积压到章末。章末钩子仍然需要，但不能是读者第一次接触到危险信号。
8. **黄金承诺**：前 3 章结束时，读者必须清楚回答"这本书讲什么"——不是世界观说明，而是"主角要做什么、为什么必须做、做不到会怎样"。

## Agent 协作架构

```
主 Agent（调度员，不读重内容）
  │
  ├─① 搜集员 Agent
  │    读 10+ 散落文件 → 组装 → 写入「上下文包-章XXX.md」磁盘
  │    返回主 agent：一句话 "上下文包已就绪"
  │
  ├─② 写手 Agent
  │    读 1 个文件（上下文包）→ 逐场景写 → 写入章节文件磁盘
  │    每场景写完即 Edit 存盘（文件=持久记忆，抗上下文压缩）
  │    返回主 agent：一句话 "初稿已完成"
  │
  └─③ 审稿人 Agent ×4（并行）
       读章节文件 + 对应教材 → 返回审稿意见
       主 agent 汇总后决定修订方案
```

## 写前准备（搜集员 Agent）

> **工作流激活**：复制 `_模板/工作流-写前准备.md` → `00-项目/工作流/写前准备-章{XXX}.md`，按追踪文件逐项执行。

**Round 0**：主 agent 读取 `00-项目/项目索引.md`（小文件，唯一一次读文件），确定：
- 本章出场角色及文件路径
- 本章涉及地点及文件路径
- 活跃伏笔、活跃情节线
- 需要哪些按需教材（伏笔/冲突/钩子等）
- 本章是否涉及特定世界观子领域（力量体系/地理/历史/社会/规则）→ 有则标注需加载的 `10-世界观/{子目录}/` 文件路径
- 检查 `00-项目/灵感池.md` 是否有与本章相关的未消费灵感（无 `[已用]` 标记的条目）→ 有则标注到搜集员 prompt 中
- 检查 `00-项目/待解决.md` 是否有与本章相关的待解决项（设定矛盾、审查 WARN 等）→ 有则标注到搜集员 prompt 中，搜集员在上下文包「本章任务」区注明需注意的约束

**Step 1**：启动搜集员 Agent
- prompt 包含：章节号、卷名、出场角色路径、地点路径、按需教材列表、需加载的世界观子目录（如有）、待解决项（如有）
- 搜集员内部并行读取所有文件，按 `_模板/上下文包模板.md` 结构组装
- 如有相关灵感条目 → 填入上下文包「灵感补充」区
- 写入 `00-项目/工作流/上下文包-章{XXX}.md`
- 从 `_模板/章节模板.md` 复制创建章节文件
- 返回主 agent："上下文包已就绪，文件路径：..."

**Step 2**：确认上下文包文件和章节文件存在 → 状态改为通过

## 写作工作流（写手 Agent）

**Step 3**：启动写手 Agent
- prompt 包含：上下文包文件路径、章节文件路径
- 写手读取 `00-项目/工作流/上下文包-章{XXX}.md`（1 个文件即拥有完整上下文）

**写手 Agent 工作方式**：

1. 读取上下文包，理解本章任务、风格约束、角色信息、前文衔接
2. 填写章节文件 frontmatter（只填必填字段；`摘要/字数` 可留空或占位，录入阶段 Batch 1 补齐）
3. （推荐：章001/新卷开篇）先写 300-500 字开篇试读段，优先满足"目标/阻碍/代价/读者问题"四项；方向确认后再继续写完整章节
4. **逐场景写入文件**（存盘协议）：
   ```
   写手必须逐场景写入文件，不得在内存中攒完整章再一次性写入：
   场景一 → Edit 写入章节文件 → 场景二 → Edit 追加 → ...
   每个场景写完后，文件已落盘，即使后续上下文被压缩也不丢失。
   ```
5. 写作过程中遵守上下文包中的防翻车指令（6 条硬约束）
6. 写完所有场景后将 frontmatter `状态` 设为 `初稿`
7. 返回主 agent："初稿已完成，X个场景，约Y字"

**Step 4**：进入审核阶段
